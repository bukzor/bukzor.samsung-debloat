# Samsung Phone Debloat - 2025-11-12

## Objective
Transform Samsung phone into a Pixel-like experience by removing bloatware and replacing Samsung apps with Google equivalents.

## Setup: ADB on Crostini

### Installing ADB
```bash
brew install android-platform-tools
```

### Wireless ADB Connection
Phone requirements (already enabled):
- Developer mode enabled
- USB debugging enabled
- Wireless debugging enabled

## Phase 2: Analysis and Discovery

### Package Inventory
Created comprehensive package analysis:

```bash
adb shell 'for pkg in $(pm list packages | cut -d: -f2); do
  echo -n "$pkg ";
  dumpsys package "$pkg" | grep firstInstallTime | head -1;
done' | sort -k2 > packages-install-dates.txt
```

Converted to structured JSON:
```bash
python3 convert_to_json.py > packages-install-dates.json
```

### Install Date Clustering
Discovered clear patterns by install date:

**Factory/System Apps** (410 packages):
- `2008-12-31` (10 packages) - Old factory timestamp
- `2021-12-31` (400 packages) - Factory ROM timestamp

**User Installs**:
- `2024-06-12` to `2024-06-16` - Initial setup burst (16 packages)
- `2025-05-11` - Major system update (56 packages)


## Lessons Learned

1. **Install date clustering** is highly effective for identifying factory bloat vs user apps
2. **`disable-user` is safer than `uninstall`** for deeply-integrated system apps
3. **Wireless ADB** works well once set up, requires same WiFi network
4. **Samsung timestamps** `2021-12-31` reliably indicate factory apps
5. **Behavior-overriding apps** (Smart features) are highest priority for removal
6. **Gboard + Lawnchair** provide excellent Pixel-like experience on Samsung

## Safety Notes

- All removals were `--user 0` only (not system-wide)
- Factory reset will restore all removed apps
- Some Samsung features (DeX, Smart features) may be wanted by some users
- Carrier config packages kept to avoid cellular issues
- Core framework and telephony packages left untouched

### Understanding SystemUI Limitations

**What we accomplished**:
- Removed Samsung's behavioral modifications and overlays
- Eliminated settings interception and usage tracking
- Removed audio/input routing overrides
- Cleared automation and customization layers

**What we cannot change without root**:
- Core SystemUI appearance (quick settings visual style)
- Notification shade design
- Status bar appearance
- System UI framework

**Root/Custom ROM required for**:
- Replacing SystemUI with AOSP/Pixel version
- Full quick settings redesign
- Complete removal of One UI framework

**Conclusion**: Achieved maximum debloat possible without root. Quick settings retain Samsung visual style, but behavior is stock Android with minimal Samsung interference.
