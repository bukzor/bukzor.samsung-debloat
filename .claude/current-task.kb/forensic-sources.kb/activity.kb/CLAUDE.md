# activity.kb — maintenance guide

Sources capturing **runtime / timeline activity** — usage, battery, process, and
scheduled-work records that show *when and what ran*. One file per capture; see
`../CLAUDE.md` for the shared schema and conventions.

Belongs here: a capture of execution/usage over time. Not here: install metadata
(→ `../packages.kb/`) or permission/role grants (→ `../capabilities.kb/`).
