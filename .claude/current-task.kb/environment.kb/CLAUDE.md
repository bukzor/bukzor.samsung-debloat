# environment.kb — maintenance guide

One file per **operational fact or gotcha** about working this task: how to
connect to the device, quirks of the Crostini/ADB toolchain, data-handling
conventions, and forensic-method caveats. Prose-only (no schema).

Belongs here: anything that, if forgotten, would cause us to re-discover a
solved problem or misread a tool's output.

Does NOT belong here: evidence-backed findings about the device
(→ `../findings.kb/`), the task plan (→ `../mission.md`), or open questions
(→ `../open-questions.kb`).

Add an entry the moment a piece of friction is resolved or a tool surprises us,
so the next agent inherits the answer instead of the problem.
