# capability-abuse.kb — maintenance guide

Analyses that **review each sensitive capability/role for abuse** — overlay,
install-unknown, accessibility, notification access, device admin, default
roles, IME, autofill. One review per capability. See `../CLAUDE.md` for the
shared schema and lifecycle.

Belongs here: "who holds capability X, and is any holder hostile?" Each consumes
the matching `../../forensic-sources.kb/capabilities.kb/` source. Not here: the
raw grants (those are sources).
