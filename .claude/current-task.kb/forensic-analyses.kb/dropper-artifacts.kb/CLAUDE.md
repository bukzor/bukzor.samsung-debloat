# dropper-artifacts.kb — maintenance guide

Analyses that work the **dropper APK and physical artifacts** — who downloaded
it, who wrote files, leftover directories, static APK analysis, signing trust.
See `../CLAUDE.md` for the shared schema and lifecycle.

Belongs here: analyses centered on files/APKs/signatures. Not here: install-time
reasoning (→ `../install-timeline.kb/`) or capability grants
(→ `../capability-abuse.kb/`).
